================================================================================
KEY ARCHITECTURAL DECISIONS
================================================================================

✓ GUI Framework: Tauri (vs Electron)
  Why: Rust-native, cross-platform, 50x smaller footprint, built-in Rust backend

✓ ML Bindings: PyO3 (vs Candle)
  Why: Reuse existing Python ecosystem; proven in production; fallback to Candle if needed

✓ Database: SQLite (vs PostgreSQL)
  Why: Embedded, zero-config, sufficient for single-instance monitoring; scale to Qdrant later

✓ Config Format: TOML (vs YAML/JSON)
  Why: Rusty idiom, human-readable, less verbose

✓ Async Runtime: Tokio (vs async-std)
  Why: Industry standard; used by actix-web, tungstenite, most ecosystem

✓ Vector DB: sqlite-vec (primary) + Qdrant (scaling)
  Why: Embedded simplicity; Qdrant option for horizontal scaling
